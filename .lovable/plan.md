
# Change document_instances.id from UUID to bigint

## Objective
Change the `public.document_instances.id` column from UUID to bigint (int8) with GENERATED BY DEFAULT, enabling direct import of 106,146 records from `unicorn1.document_instances` with preserved original IDs.

---

## Current State

| Attribute | Current Value |
|-----------|---------------|
| id type | UUID |
| id default | gen_random_uuid() |
| rows | 0 |
| FK constraints | None |

---

## Target State

| Attribute | New Value |
|-----------|-----------|
| id type | bigint (int8) |
| id default | GENERATED BY DEFAULT AS IDENTITY |
| rows | Ready for 106,146 imports |

---

## Migration Strategy

Since the table is empty and has no foreign key dependencies, we will:

1. **Drop the existing table**
2. **Recreate with correct schema** including:
   - `id` as bigint GENERATED BY DEFAULT AS IDENTITY
   - `stage_instance_id` (bigint) to link to stage_instances
   - All columns aligned with unicorn1 structure

---

## SQL Migration

```sql
-- Drop existing empty table
DROP TABLE IF EXISTS public.document_instances;

-- Recreate with bigint id and proper structure
CREATE TABLE public.document_instances (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  document_id bigint NOT NULL,
  stage_instance_id bigint,
  tenant_id bigint,
  is_generated boolean DEFAULT false,
  generation_date timestamptz,
  status text,
  notes text,
  created_at timestamptz DEFAULT now() NOT NULL,
  updated_at timestamptz DEFAULT now() NOT NULL
);

-- Add RLS
ALTER TABLE public.document_instances ENABLE ROW LEVEL SECURITY;

-- Add index on stage_instance_id for efficient joins
CREATE INDEX idx_document_instances_stage_instance 
  ON public.document_instances(stage_instance_id);

-- Add index on tenant_id
CREATE INDEX idx_document_instances_tenant 
  ON public.document_instances(tenant_id);
```
