# EOS Quarterly Conversations (QC)

## Purpose

Quarterly Conversations are private, one-on-one strategic alignment sessions between managers and team members. They ensure everyone is in the Right Seat™, aligned with core values, and has a clear development plan for continued growth.

## EOS Alignment

QC sessions align with core EOS principles:
- **Core Values**: Assess adherence to organizational values
- **Right Person, Right Seat™**: Evaluate using GWC (Get it, Want it, Capacity)
- **Rocks**: Review quarterly goals and carryover planning
- **Issues**: Identify challenges for IDS discussion
- **Accountability**: Clear expectations and development actions

## Participants & Roles

### Reviewee (Team Member)
- Subject of the quarterly review
- Can view and edit their own QC
- Must sign off to complete the conversation
- Receives automatic notifications and reminders

### Manager (Facilitator)
- Conducts the conversation
- Has full edit access during the session
- Must sign off to complete the conversation
- Can schedule follow-up conversations

### HR/Admin
- Can view all QCs within their tenant
- Cannot edit content (read-only access)
- Useful for organizational development oversight

### SuperAdmin
- Full access to all QCs across all tenants
- Administrative oversight and audit purposes

## Privacy & Security

**Critical**: Quarterly Conversations contain sensitive performance and development information.

### Access Control
- Only accessible by: Reviewee, assigned Manager(s), HR/Admin, SuperAdmin
- **Never visible to clients** - internal use only
- Strict tenant isolation enforced via RLS policies
- After both parties sign, conversation becomes read-only

### Data Sensitivity
- Performance assessments
- Career development discussions
- Core values alignment feedback
- Personal development plans
- Confidential notes and observations

## Default Template Sections

The standard "EOS Quarterly Conversation" template includes:

### 1. Core Values Assessment
Rate alignment with each organizational core value:
- Scale: Rarely / Sometimes / Consistently
- Free-form notes for examples and discussion

### 2. GWC (Get it, Want it, Capacity)
Assess seat fit across three dimensions:
- **Gets It**: Understands role and what success looks like
- **Wants It**: Passionate, motivated, and engaged
- **Capacity**: Has time, capability, and resources
- Discussion notes

### 3. Seat Expectations & Roles
- Review accountability chart role
- Rate performance: Needs Work / Meets / Exceeds
- Discussion of role clarity and execution

### 4. Rocks Review
- List current quarter rocks and status
- Identify rocks to carry over to next quarter
- Discuss obstacles and support needed

### 5. Keep / Start / Stop
Three-column retrospective:
- **Keep**: What's working well
- **Start**: New opportunities or behaviors
- **Stop**: What to eliminate or change

### 6. Wins & Headlines
- Recent accomplishments and successes
- Other important updates or headlines

### 7. Issues for IDS
- Capture issues requiring deeper discussion
- Links to EOS Issues list for IDS process

### 8. Development Plan & Actions
- Create specific action items and development goals
- Links to EOS To-Dos for tracking
- Set focus areas for next quarter

## Workflow

### 1. Scheduling
1. Manager or Admin navigates to **EOS → Quarterly Conversations**
2. Click "Schedule QC"
3. Select:
   - Team Member (reviewee)
   - Manager (facilitator)
   - Quarter start date (end date auto-calculates)
   - Scheduled meeting date/time
4. Default EOS template is automatically assigned
5. Both parties receive calendar invite and notifications

### 2. Conducting the Session
1. Navigate to the specific QC from the list
2. Work through each section tab
3. **Auto-save**: All answers save automatically after 1-second delay
4. Manager can guide discussion through each section
5. Both parties can edit during the session
6. Create action items (Issues, To-Dos, Rocks) as needed

### 3. Section-by-Section Completion
- Move through tabs in order
- Complete required fields (marked with *)
- Add context and notes throughout
- GWC section has visual indicator when all three are "Yes"

### 4. Sign-off Process
1. Navigate to "Summary & Sign-off" tab
2. Review all completed sections
3. **Manager signs first** (recommended)
4. **Reviewee signs second**
5. After both signatures:
   - Conversation locked (read-only)
   - Status changes to "completed"
   - PDF summary can be generated
   - "Schedule Next QC" becomes available

### 5. Follow-up Actions
- Action items appear in respective EOS modules (Issues, To-Dos, Rocks)
- Schedule next quarter's conversation (auto-calculates dates)
- Export PDF for records
- Calendar integration for recurring reminders

## Integration with Other EOS Modules

### Accountability Chart
- Pull seat/role information for Seat Expectations section
- Link QC to specific chart positions
- Inform organizational structure decisions

### Rocks
- Review current quarter rocks progress
- Create next quarter rocks from development plan
- Mark rocks for carryover

### Issues
- Create issues from QC discussion
- Link to existing issues for context
- Prioritize for IDS sessions

### To-Dos
- Convert development actions into trackable to-dos
- Assign owners and due dates
- Track completion through standard To-Dos module

## Notifications & Reminders

Automatic notifications sent via email and in-app:
- **1 week before**: Reminder to schedule/prepare
- **24 hours before**: Final reminder
- **At completion**: Confirmation to both parties
- **Follow-up**: Reminder if next QC not scheduled

## Best Practices

### Preparation
- Schedule in a quiet, private space
- Review previous QC (if available)
- Prepare current quarter rocks status
- Have accountability chart handy

### During Conversation
- Start with wins and positives
- Be honest but constructive
- Focus on future development
- Document specific examples
- Create actionable items, not vague goals

### After Conversation
- Both parties sign promptly
- Review and accept linked action items
- Schedule next QC immediately
- Follow up on development actions regularly

## Retention & Compliance

- QC records retained indefinitely for employee development history
- Full audit trail of all changes
- PDF snapshots preserve conversation state at sign-off
- Access logs for compliance and security

## Troubleshooting

### Can't see a QC
- Verify you are either the reviewee, manager, or admin
- Check that QC belongs to your tenant
- Confirm not filtered out by tab selection

### Can't edit answers
- QC may be signed off (read-only after both sign)
- Verify you have access (reviewee or manager)
- Check network connection for auto-save

### Signature fails
- Must be correct role (manager or reviewee)
- Cannot sign twice
- Must have access to the QC

### Action items not appearing
- Verify items created via "Create Links" functionality
- Check respective EOS module (Issues, To-Dos, Rocks)
- Confirm tenant_id matches
